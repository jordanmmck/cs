~~~ site-packages
Anything installed in site-packages can be imported by python programs.

~~~ virtualenv
A virtual environment is just a directory that contains a complete copy of everything needed to run python programs. Including a copy of the python binary, standard lib, pip, and site-packages.
virtualenv puts a copy of pip into
pip and virtualenv are the only packages that ever need to be installed globally.

The env directory at project root will contain stuff like bin, include, and lib. 
`bin`: where python binary lives and pip
`source env/bin/activate`: activates virtual env, now do pip install whatever and it will install locally.
